SKILLS

Six skills, one shared language.

Every skill inherits the same design tokens — colors, type, motion, radii. What changes is layout, components, and the quality bar for each product mode.

design-tokens

The shared foundation: Google Sans typography, blue-600 accent, 24px radii, soft shadows, spring motion. Read first — values are law.

landing

Marketing pages with mesh gradients, floating orbs, glass nav, animated hero mockups. References: Google Antigravity, Gemini.

desktop

Native-feeling apps with macOS traffic lights, sidebars, command palettes. Built for Tauri / Electron / IDE extensions.

saas

Multi-page business apps. App shell, data tables, settings, forms, dashboards. References: Stripe, Linear, Notion.

mobile

Touch-first apps with 44px tap targets, bottom tabs, device frames, safe areas. References: iOS HIG, Material You.

docs

Knowledge bases with 3-col layouts, 720px content, code blocks, API reference, prev/next. References: GitBook, Stripe Docs.

GALLERY

Five demos. Real multi-page apps.

Each demo is a complete multi-page project — three real screens per mode, sharing the same design tokens. Open any case and read the source.

WORKFLOW

Read tokens. Pick mode. Ship UI.

01

Load design tokens

AI assistant reads skills/design-tokens/SKILL.md. Colors, type, motion, radii — all defined as Tailwind classes that get baked into every output.

02

Detect mode, load 1 skill

Based on your prompt (landing? dashboard? mobile app?), exactly one mode skill loads. No mode stacking, no guessing. Each mode owns its layout vocabulary.

03

Run 26-point quality gate

Every UI is checked against 26 criteria across design, system, experience, content, accessibility, anti-patterns. Any failure = redo. No "looks fine, ship it".
